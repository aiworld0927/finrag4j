# 48_运维监控指南

## 1. 监控体系

### 1.1 监控架构

```
采集层 → 存储层 → 分析层 → 展示层
  |          |          |         |
Metrics   InfluxDB   PromQL   Grafana
Logs      Elasticsearch Kibana
Traces    Jaeger
```

### 1.2 监控指标

| 分类 | 指标 | 阈值 |
| :--- | :--- | :--- |
| 主机 | CPU使用率 | > 80%告警 |
| 主机 | 内存使用率 | > 85%告警 |
| 主机 | 磁盘使用率 | > 90%告警 |
| 应用 | 响应时间 | > 2s告警 |
| 应用 | 请求成功率 | < 99%告警 |
| 应用 | 错误率 | > 1%告警 |

## 2. 日志管理

### 2.1 日志规范

```
[时间] [级别] [模块] [线程] [类名:行号] - 日志内容
2024-01-01 10:00:00 INFO  auth-service [main] UserService:45 - 用户登录成功: admin
```

### 2.2 日志级别

| 级别 | 使用场景 |
| :--- | :--- |
| DEBUG | 详细调试信息，生产环境关闭 |
| INFO | 正常业务流程记录 |
| WARN | 警告信息，需要关注 |
| ERROR | 错误信息，需要处理 |

### 2.3 日志收集

- 使用ELK栈收集和分析日志
- 日志按服务分类存储
- 支持按时间、级别、关键词搜索

## 3. 告警配置

### 3.1 告警规则

| 告警类型 | 条件 | 通知方式 |
| :--- | :--- | :--- |
| 服务宕机 | 心跳检测失败 | 短信 + 钉钉 |
| 高CPU | CPU > 90%持续5分钟 | 钉钉 |
| 高内存 | 内存 > 90%持续5分钟 | 钉钉 |
| 服务错误 | 错误率 > 5% | 短信 + 钉钉 |
| 数据库连接 | 连接池使用率 > 90% | 钉钉 |

### 3.2 告警通知

- 钉钉群通知
- 短信通知（紧急告警）
- 邮件通知（日报）

## 4. 性能监控

### 4.1 JVM监控

- Heap内存使用
- GC次数和时间
- 线程数量
- 类加载数量

### 4.2 数据库监控

- 慢查询日志
- 连接池状态
- 查询响应时间
- 锁等待时间

### 4.3 接口监控

- 接口响应时间分布
- 接口调用次数
- 接口成功率
- 接口错误率

## 5. 健康检查

### 5.1 健康检查端点

| 端点 | 用途 |
| :--- | :--- |
| /actuator/health | 健康状态 |
| /actuator/info | 应用信息 |
| /actuator/metrics | 指标数据 |
| /actuator/prometheus | Prometheus指标 |

### 5.2 健康状态

- UP：服务正常
- DOWN：服务异常
- OUT_OF_SERVICE：服务不可用

## 6. 容量规划

- 定期评估资源使用情况
- 根据业务增长预测容量需求
- 提前进行扩容准备
- 定期进行压测验证
