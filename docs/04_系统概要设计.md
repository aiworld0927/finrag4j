# 4_系统概要设计

## 1. 系统架构

### 1.1 架构设计原则
- 微服务架构
- 模块化设计
- 高可用性
- 可扩展性

### 1.2 技术选型
- 语言：Java 21 + Python 3.10
- 框架：SpringBoot 3.2 + FastAPI
- 数据库：PostgreSQL + PGVector + Redis
- 消息队列：RocketMQ
- 对象存储：MinIO

### 1.3 模块划分
- gateway：API网关
- auth：认证授权
- document：文档服务
- search：检索服务
- agent：Agent服务
- python-service：预处理服务

---

## 2. 模块职责划分

### 2.1 API网关模块（gateway）

#### 2.1.1 模块职责
作为系统的统一入口，负责请求路由、负载均衡、认证鉴权、限流熔断等。

#### 2.1.2 核心功能
- **请求路由**：根据请求路径将请求转发到对应的微服务
- **负载均衡**：在多个服务实例之间分配请求，提高系统可用性
- **认证鉴权**：验证用户身份和权限，拦截未授权请求
- **限流熔断**：防止系统过载，保护后端服务
- **日志记录**：记录所有请求和响应日志，用于审计和监控
- **跨域处理**：处理跨域请求，支持前后端分离部署

#### 2.1.3 技术实现
- Spring Cloud Gateway
- Redis实现分布式限流
- JWT Token验证
- Sentinel实现熔断降级

---

### 2.2 认证授权模块（auth）

#### 2.2.1 模块职责
负责用户身份认证、授权管理、单点登录、权限控制等安全相关功能。

#### 2.2.2 核心功能
- **用户认证**：支持用户名密码、手机验证码、第三方登录等多种认证方式
- **单点登录**：支持CAS、OAuth2等单点登录协议
- **权限管理**：基于RBAC的权限控制，支持角色和权限的灵活配置
- **Token管理**：JWT Token的生成、验证、刷新
- **审计日志**：记录用户登录、权限变更等安全相关操作
- **多租户支持**：支持多租户数据隔离

#### 2.2.3 技术实现
- Spring Security
- JWT
- CAS Client
- OAuth2

---

### 2.3 文档服务模块（document）

#### 2.3.1 模块职责
负责文档的上传、存储、解析、管理、版本控制等全生命周期管理。

#### 2.3.2 核心功能
- **文档上传**：支持批量上传、断点续传、大文件分片上传
- **文档存储**：将文档存储到MinIO对象存储
- **文档解析**：调用Python预处理服务进行文档解析
- **文档管理**：文档的增删改查、分类、标签管理
- **版本控制**：支持文档版本管理和历史版本回溯
- **文档检索**：支持文档的元数据检索和全文检索
- **权限控制**：文档级别的权限控制，控制文档的访问权限

#### 2.3.3 技术实现
- SpringBoot
- MinIO Client
- RestTemplate调用Python服务
- PostgreSQL存储文档元数据

---

### 2.4 检索服务模块（search）

#### 2.4.1 模块职责
负责文本向量化、向量存储、相似度检索、结果排序等检索相关功能。

#### 2.4.2 核心功能
- **文本向量化**：调用大模型API将文本转换为向量
- **向量存储**：将向量存储到PGVector数据库
- **相似度检索**：基于向量相似度进行语义检索
- **混合检索**：结合向量检索和关键词检索，提高检索准确率
- **结果排序**：根据相似度、相关性、时间等因素对结果排序
- **结果过滤**：支持按文档类型、时间、来源等条件过滤结果
- **结果高亮**：对检索结果中的关键词进行高亮显示

#### 2.4.3 技术实现
- LangChain4j
- PostgreSQL + PGVector
- Redis缓存热点数据
- 大模型API（OpenAI、通义千问等）

---

### 2.5 Agent服务模块（agent）

#### 2.5.1 模块职责
负责智能问答、Agent流程编排、工具调用、多轮对话等智能化功能。

#### 2.5.2 核心功能
- **智能问答**：基于RAG技术，结合检索结果生成准确回答
- **Agent流程编排**：支持可视化流程编排，配置复杂的业务流程
- **工具调用**：支持调用计算器、API等外部工具
- **多轮对话**：保持对话上下文，支持连续提问
- **流程分支**：支持条件判断和分支逻辑
- **任务链**：支持多步骤任务编排，前一步的输出作为后一步的输入
- **回答溯源**：回答可追溯到具体的文档来源，增强可信度

#### 2.5.3 技术实现
- LangChain4j
- SpringBoot
- RocketMQ异步任务处理
- 大模型API

---

### 2.6 预处理服务模块（python-service）

#### 2.6.1 模块职责
负责文档解析、OCR识别、文本清洗、文本分块等预处理任务。

#### 2.6.2 核心功能
- **文档解析**：支持PDF、Word、Excel、TXT等多种文档格式的解析
- **OCR识别**：基于PaddleOCR实现离线中文OCR识别，支持图片和扫描PDF
- **文本清洗**：去除停用词、特殊字符、多余空格等
- **文本分块**：针对金融文档特点设计三种分块策略
  - 监管文件：600字符/块，重叠100字符
  - 信贷合同：800字符/块，重叠150字符
  - 内部通知：400字符/块，重叠50字符
- **文本规范化**：统一文本格式，包括标点、数字、日期等
- **表格识别**：识别文档中的表格，提取表格数据

#### 2.6.3 技术实现
- FastAPI
- PaddleOCR
- python-docx、openpyxl、pdfplumber
- jieba分词
- Uvicorn服务器

#### 2.6.4 设计原则
- **无状态**：服务无状态，支持水平扩展
- **无数据库**：不直接访问数据库，通过HTTP接口与Java服务交互
- **无业务逻辑**：只提供工具能力，不包含业务逻辑
- **高并发**：支持高并发请求，使用异步IO

文档上传 → 解析/OCR → 分块 → 向量化 → 存储 → 检索 → 回答

### 2.2 数据库设计
- document：文档信息表
- chunk：文本块表
- vector：向量数据表
- user：用户表
- role：角色表

### 2.3 安全设计
- OAuth2 + JWT认证
- RBAC权限控制
- API限流
- 审计日志
