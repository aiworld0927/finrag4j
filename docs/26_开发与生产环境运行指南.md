## 1. 开发环境运行

### 1.1 环境准备

**基础环境要求**：

| 软件 | 版本 | 说明 |
|------|------|------|
| JDK | 21+ | Java开发环境 |
| Python | 3.12+ | Python运行环境 |
| Maven | 3.9+ | Java依赖管理 |
| uv | 0.1+ | Python依赖管理 |
| Docker | 24+ | 容器运行时 |
| Node.js | 20+ | 前端开发 |

**开发工具**：
- IDE：IntelliJ IDEA / VS Code
- 数据库客户端：DBeaver / DataGrip

### 1.2 后端服务开发模式

#### 1.2.1 启动中间件

```bash
# 进入deploy目录
cd deploy

# 启动基础中间件（Nacos、PostgreSQL、Redis、MinIO、RocketMQ）
docker-compose -f docker-compose-base.yml up -d

# 检查中间件状态
docker-compose -f docker-compose-base.yml ps
```

#### 1.2.2 启动微服务

**方式一：使用IDE启动（推荐）**

在IntelliJ IDEA中依次启动：

1. `finrag4j-gateway` - 端口 8080
2. `finrag4j-auth` - 端口 8081
3. `finrag4j-document` - 端口 8082
4. `finrag4j-search` - 端口 8083
5. `finrag4j-agent` - 端口 8084

**方式二：使用Maven命令启动**

```bash
# 进入backend目录
cd backend

# 编译所有模块
mvn clean compile -DskipTests

# 启动Gateway
cd finrag4j-gateway
mvn spring-boot:run

# 启动Auth服务（新开终端）
cd ../finrag4j-auth
mvn spring-boot:run

# 启动Document服务（新开终端）
cd ../finrag4j-document
mvn spring-boot:run

# 启动Search服务（新开终端）
cd ../finrag4j-search
mvn spring-boot:run

# 启动Agent服务（新开终端）
cd ../finrag4j-agent
mvn spring-boot:run
```

#### 1.2.3 开发配置

开发环境使用 `bootstrap.yml` 中的配置：

```yaml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: ${NACOS_HOST:localhost}:8848
      config:
        server-addr: ${NACOS_HOST:localhost}:8848
```

### 1.3 Python服务开发模式

#### 1.3.1 环境准备

```bash
# 进入Python服务目录
cd backend/finrag4j-python

# 安装依赖（自动创建.venv）
uv sync

# 激活虚拟环境
source .venv/bin/activate  # Linux/Mac
.venv\Scripts\activate     # Windows
```

#### 1.3.2 启动方式

**开发模式（自动重载）**：

```bash
# 方式一：直接运行
uv run python main.py

# 方式二：启用调试模式
DEBUG_MODE=true uv run python main.py

# 方式三：禁用Nacos（纯本地开发）
NACOS_ENABLED=false DEBUG_MODE=true uv run python main.py

# 方式四：使用Uvicorn
uvicorn main:app --host 0.0.0.0 --port 8001 --reload
```

#### 1.3.3 开发环境配置

| 环境变量 | 说明 | 开发值 |
|----------|------|--------|
| DEBUG_MODE | 调试模式 | true |
| NACOS_ENABLED | 启用Nacos | false（本地开发） |
| LOG_LEVEL | 日志级别 | DEBUG |
| SERVER_PORT | 服务端口 | 8001 |

### 1.4 前端开发模式

```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install

# 开发模式启动（热重载）
npm run dev

# 构建生产版本
npm run build
```

### 1.5 开发环境配置文件

创建 `backend/.env` 文件：

```env
# 开发环境配置
NACOS_HOST=localhost
NACOS_PORT=8848
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
REDIS_HOST=localhost
REDIS_PORT=6379
MINIO_HOST=localhost
MINIO_PORT=9000
ROCKETMQ_NAMESRV_ADDR=localhost:9876
```

## 2. 生产环境运行

### 2.1 Docker Compose部署

#### 2.1.1 一键部署

```bash
# 进入deploy目录
cd deploy

# 启动所有服务（包含微服务和中间件）
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看服务日志
docker-compose logs -f [service-name]
```

#### 2.1.2 分阶段部署

```bash
# 第一步：启动基础中间件
docker-compose -f docker-compose-base.yml up -d

# 等待中间件启动完成（约30秒）
sleep 30

# 第二步：启动微服务
docker-compose -f docker-compose.yml up -d
```

#### 2.1.3 停止服务

```bash
# 停止所有服务
docker-compose down

# 停止并删除数据卷（谨慎使用）
docker-compose down -v
```

### 2.2 环境变量配置

生产环境使用 `deploy/.env` 文件：

```env
# Nacos配置
NACOS_HOST=nacos
NACOS_PORT=8848
NACOS_NAMESPACE=public

# 数据库配置
POSTGRES_HOST=postgres
POSTGRES_PORT=5432
POSTGRES_DB=finrag4j
POSTGRES_USER=finrag4j_user
POSTGRES_PASSWORD=your_secure_password

# Redis配置
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=your_redis_password

# MinIO配置
MINIO_ROOT_USER=admin
MINIO_ROOT_PASSWORD=your_minio_password

# 服务端口配置
GATEWAY_PORT=8080
PYTHON_SERVICE_PORT=8001
```

### 2.3 服务启动验证

```bash
# 检查Gateway状态
curl http://localhost:8080/actuator/health

# 检查Python服务状态
curl http://localhost:8001/health

# 检查Nacos服务注册
curl http://localhost:8848/nacos/v1/ns/instance/list?serviceName=finrag4j-gateway

# 查看所有服务日志
docker-compose logs --tail=50
```

## 4. 常用命令速查

### 4.1 Docker Compose命令

```bash
# 启动所有服务
docker-compose up -d

# 启动指定服务
docker-compose up -d finrag4j-gateway finrag4j-auth

# 停止所有服务
docker-compose down

# 重启服务
docker-compose restart [service-name]

# 查看日志
docker-compose logs -f [service-name]

# 查看服务状态
docker-compose ps

# 进入容器
docker-compose exec [service-name] bash
```

### 4.2 Maven命令

```bash
# 编译所有模块
mvn clean compile -DskipTests

# 打包所有模块
mvn clean package -DskipTests

# 运行指定服务
mvn spring-boot:run

# 运行测试
mvn test
```

### 4.3 Python命令

```bash
# 安装依赖
uv sync

# 添加依赖
uv add [package-name]

# 启动开发模式
uv run python main.py

# 启动生产模式
uvicorn main:app --host 0.0.0.0 --port 8001
```

### 4.4 日志查看

```bash
# 查看Gateway日志
docker-compose logs -f finrag4j-gateway

# 查看Python服务日志
docker-compose logs -f finrag4j-python

# 查看最近100条日志
docker-compose logs --tail=100 finrag4j-auth
```

## 附录：启动顺序建议

### 开发环境启动顺序

1. **启动中间件**（docker-compose-base.yml）
2. **启动Nacos**（等待约10秒）
3. **启动微服务**（按Gateway → Auth → Document → Search → Agent顺序）
4. **启动Python服务**
5. **启动前端**

### 生产环境启动顺序

```bash
# 1. 启动基础中间件
docker-compose -f docker-compose-base.yml up -d

# 2. 等待中间件就绪
sleep 30

# 3. 启动微服务
docker-compose -f docker-compose.yml up -d

# 4. 验证所有服务
docker-compose ps
```

## 注意事项

1. **端口冲突**：确保8080、8081、8082、8083、8084、8001端口未被占用
2. **内存要求**：建议至少16GB内存用于开发环境，32GB用于生产环境
3. **首次启动**：OCR模型首次运行会自动下载，需要联网环境
4. **Nacos配置**：生产环境建议配置持久化存储
5. **日志管理**：生产环境建议配置日志轮转和集中日志收集
