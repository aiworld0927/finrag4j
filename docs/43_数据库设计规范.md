# 43_数据库设计规范

## 1. 设计原则

### 1.1 命名规范

| 对象类型 | 命名规则 | 示例 |
| :--- | :--- | :--- |
| 数据库 | `{项目缩写}_{服务名}` | `finrag4j_auth` |
| 表名 | 小写英文，下划线分隔，前缀服务名 | `auth_user`, `doc_document` |
| 字段名 | 小写英文，下划线分隔 | `user_id`, `created_at` |
| 索引名 | `idx_{表名}_{字段名}` | `idx_user_email` |
| 外键名 | `fk_{子表名}_{父表名}_{字段名}` | `fk_order_user_user_id` |

### 1.2 字段设计规范

- **主键**：统一使用 `{表名}_id`，类型 BIGINT，自增
- **创建时间**：`created_at`，类型 TIMESTAMP，默认 CURRENT_TIMESTAMP
- **更新时间**：`updated_at`，类型 TIMESTAMP，默认 CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
- **逻辑删除**：`deleted`，类型 TINYINT，默认 0，1 表示已删除
- **枚举字段**：使用 TINYINT 或 VARCHAR(32)，配合枚举类使用

## 2. 表结构设计

### 2.1 通用字段

```sql
CREATE TABLE example_table (
    example_id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    deleted TINYINT DEFAULT 0 COMMENT '逻辑删除标识',
    created_by BIGINT COMMENT '创建人ID',
    updated_by BIGINT COMMENT '更新人ID'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='示例表';
```

### 2.2 索引设计

- 主键索引：必须创建
- 唯一索引：用于唯一约束字段（如邮箱、手机号）
- 普通索引：用于查询条件字段
- 复合索引：根据查询场景合理创建，遵循最左前缀原则

## 3. 数据类型选择

| 数据类型 | 使用场景 | 注意事项 |
| :--- | :--- | :--- |
| VARCHAR | 变长字符串 | 最大长度按实际需求设置，避免使用 TEXT |
| INT | 整数类型 | 普通ID使用 INT，大表使用 BIGINT |
| BIGINT | 大整数类型 | 主键、外键优先使用 |
| DECIMAL | 精确数值 | 金额字段使用 DECIMAL(18,4) |
| TIMESTAMP | 时间戳 | 存储日期时间，时区敏感 |
| DATE | 日期 | 仅存储日期部分 |

## 4. 关系设计

### 4.1 一对一关系
使用外键关联，外键字段设置唯一约束

### 4.2 一对多关系
在多方表中设置外键字段，指向一方主键

### 4.3 多对多关系
创建中间表，包含两个外键字段

## 5. 约束设计

- **非空约束**：必填字段设置 NOT NULL
- **唯一约束**：唯一标识字段设置 UNIQUE
- **外键约束**：关联字段设置 FOREIGN KEY，ON DELETE CASCADE/SET NULL
- **检查约束**：使用业务代码或触发器实现

## 6. 性能优化

- 合理创建索引，避免索引过多
- 使用分区表处理大数据量
- 避免使用 SELECT *，只查询需要的字段
- 使用批量操作减少数据库交互
