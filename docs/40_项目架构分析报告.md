## 文档概述

本文档对 FinRag4j 项目进行全面的架构分析与梳理，识别功能缺失点、不准确之处，并提供架构改进建议。

**文档版本**: v1.1.0  
**编制日期**: 2026-06-22  
**分析范围**: 功能架构、集成架构、部署架构、交互关系

## 1. 项目概述

### 1.1 项目定位

FinRag4j 是一款**面向金融行业的企业级大模型RAG应用框架**，专注于金融文档处理、智能问答、合规分析等场景。

### 1.2 核心技术栈

| 分类 | 技术 | 版本 |
|------|------|------|
| 后端框架 | Spring Boot + Spring Cloud | 3.2.x / 2023.0.x |
| 服务注册/配置 | Nacos | 2.2.x |
| RAG框架 | LangChain4j | 0.30+ |
| 数据库 | PostgreSQL + PGVector | 16+ |
| 缓存 | Redis | 7.x |
| 消息队列 | RocketMQ | 5.0+ |
| 对象存储 | MinIO | 2024+ |
| Python服务 | FastAPI + PaddleOCR | 0.110+ / 2.8+ |
| 前端 | Vue3 + Element Plus | 3.4+ / 2.6+ |

## 5. 中间件地址管理

### 5.1 配置管理机制

项目采用**多级配置管理**机制：

```
┌─────────────────────────────────────────────────────────────────────┐
│                        第1级: .env 文件                              │
│              (环境变量配置，位于 deploy/ 目录)                         │
│   NACOS_HOST, POSTGRES_HOST, REDIS_HOST, MINIO_ENDPOINT 等         │
└─────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────┐
│                     第2级: Docker Compose                             │
│                 (docker-compose.yml / docker-compose-base.yml)        │
│              使用 ${VAR} 引用 .env 中的环境变量                        │
│              使用 depends_on + condition: service_healthy             │
└─────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────┐
│                     第3级: 应用配置文件                               │
│            (bootstrap.yml - Nacos连接, application.yml - 业务配置)      │
│                  使用 ${VAR:default} 支持默认值                       │
└─────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────┐
│                     第4级: Nacos 配置中心                             │
│                  (动态配置，服务间共享配置)                              │
│              finrag4j-common.yml 等 shared-configs                    │
└─────────────────────────────────────────────────────────────────────┘
```

## 7. 不准确之处说明

### 7.1 端口配置不一致

#### 7.1.1 Nacos 端口说明（Nacos 3.2 版本）

**Nacos 3.2 版本端口设计**:
- **8848 端口**: 程序访问端口，用于服务注册/发现和配置中心 API 调用
- **8080 端口**: 管理控制台端口，用于 Web UI 访问

| 文件 | 端口配置 | 说明 |
|------|----------|------|
| .env | NACOS_PORT=8848 | 程序访问端口 |
| docker-compose-base.yml | 8848:8848, 8080:8080 | 双重映射，分别对应程序端口和控制台端口 |
| bootstrap.yml | NACOS_PORT:8848 | 程序连接使用 8848 |

**说明**: 
- Docker Compose 中同时映射 8848 和 8080 两个端口是**正确的设计**，而非问题
- **8848**: 供微服务程序调用 Nacos API 使用
- **8080**: 供运维人员访问 Nacos 管理控制台使用

**访问方式**:
- Nacos 管理控制台: `http://{NACOS_HOST}:8080/nacos`
- Nacos API/SDK 连接: `{NACOS_HOST}:8848`

#### 7.1.2 Agent 服务端口问题

| 文件 | 端口配置 | 说明 |
|------|----------|------|
| docker-compose.yml | 8086:8084 | 外部8086，内部8084 |
| .env | AGENT_PORT=8086 | 环境变量 |
| bootstrap.yml | 无 | 服务本身监听8084 |
