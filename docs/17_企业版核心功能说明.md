## 2. RBAC权限 & 多租户管理

### 2.1 用户管理

支持完整的用户生命周期管理：

- 创建/编辑/删除用户
- 用户状态管理（启用/禁用）
- 密码加密存储（BCrypt）
- 登录验证

### 2.2 角色管理

预设5种系统角色：

| 角色 | 编码 | 说明 |
|------|------|------|
| 超级管理员 | super_admin | 系统最高权限 |
| 知识库管理员 | kb_admin | 知识库管理权限 |
| 合规审核 | compliance_audit | 合规审核权限 |
| 客户经理 | account_manager | 客户服务权限 |
| 只读用户 | read_only | 只读访问权限 |

### 2.3 权限层级

三级权限控制：

```
┌─────────────────────────────────────┐
│  系统权限层  │ sys:manage, sys:user:*  │
├─────────────────────────────────────┤
│  知识库权限层 │ kb:view, kb:edit       │
├─────────────────────────────────────┤
│  Agent权限层 │ agent:credit_extract   │
└─────────────────────────────────────┘
```

### 2.4 多租户隔离（商业版）

**物理隔离方案**：
- 独立向量空间
- 独立知识库
- 独立数据库表（通过tenant_id区分）
- 独立算力配额

**租户配额配置**：
- 最大用户数
- 最大知识库数
- 日请求限额
- 最大并发数
- 模型白名单

### 2.5 接口清单

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/users` | POST/GET | 用户CRUD |
| `/api/users/login` | POST | 用户登录 |
| `/api/users/{id}/roles` | GET/POST | 角色分配 |
| `/api/roles` | POST/GET | 角色CRUD |
| `/api/roles/{id}/permissions` | GET/POST | 权限分配 |

## 3. 全链路不可篡改审计日志

### 3.1 审计范围

AOP切面自动捕获以下操作：

- 文档操作（上传、删除、预览）
- 问答对话
- Agent执行
- 权限修改
- 接口调用

### 3.2 日志结构

| 字段 | 说明 |
|------|------|
| tenant_id | 租户ID |
| user_id | 用户ID |
| operation_type | 操作类型 |
| operation_module | 操作模块 |
| request_url | 请求URL |
| request_params | 请求参数 |
| response_data | 响应数据 |
| ip_address | IP地址 |
| execution_time | 执行时间(ms) |
| success | 是否成功 |

### 3.3 日志留存策略

- 默认保留5年
- 可配置保留年限
- 支持自动清理过期日志

### 3.4 报表导出

支持导出监管报表：

- Excel格式
- PDF格式（扩展）
- 支持按用户/时间/租户筛选

### 3.5 接口清单

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/audit/time-range` | GET | 按时间范围查询 |
| `/api/audit/user/{userId}` | GET | 按用户查询 |
| `/api/audit/export/excel` | GET | 导出Excel |
| `/api/audit/clean` | DELETE | 清理过期日志 |

## 4. 金融垂直Agent业务

### 4.1 信贷材料抽取Agent

**功能特性**：
- 自定义抽取模板
- 批量合同字段提取
- 结构化导出Excel
- 人工修正记录

**抽取流程**：

```
上传合同文档 → 调用Python解析 → LLM结构化抽取 → 存储结果 → 人工复核
```

**接口清单**：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/agent/extract/template` | POST | 创建抽取模板 |
| `/api/agent/extract/execute` | POST | 执行抽取 |
| `/api/agent/extract/{id}/review` | POST | 人工复核 |

### 4.2 监管合规自查Agent

**功能特性**：
- 业务材料匹配监管知识库
- 风险分级（低/中/高/严重）
- 自动生成整改报告
- 强制人工复核节点

**合规检查流程**：

```
选择文档 → 匹配监管条款 → 风险评估 → 生成报告 → 人工复核
```

**风险等级定义**：

| 等级 | 描述 | 处理方式 |
|------|------|----------|
| low | 轻微问题 | 记录观察 |
| medium | 一般问题 | 需要整改 |
| high | 严重问题 | 立即整改 |
| critical | 重大问题 | 暂停业务 |

**接口清单**：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/agent/compliance/report` | POST | 创建自查报告 |
| `/api/agent/compliance/check` | POST | 执行合规检查 |
| `/api/agent/compliance/report/{id}/review` | POST | 复核报告 |

### 4.3 制度咨询Agent

**功能特性**：
- 制度条款检索
- 智能问答
- 法规依据引用

**接口清单**：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/agent/regulation/query` | POST | 制度咨询 |
| `/api/agent/regulation/clause` | POST | 添加条款 |

### 4.4 客户经理营销方案生成Agent（基础逻辑）

**功能特性**：
- 客户画像分析
- 产品匹配推荐
- 营销方案生成

## 5. 可视化Agent工作流引擎

### 5.1 工作流定义

支持的节点类型：

| 节点类型 | 说明 |
|----------|------|
| start | 开始节点 |
| end | 结束节点 |
| task | 自动任务 |
| approval | 人工审批 |
| review | 复核节点 |
| branch | 分支节点 |

### 5.2 流程执行模式

**手动触发**：
- API调用触发
- 即时执行

**定时触发**：
- Cron表达式配置
- 自动定时执行

### 5.3 流程逻辑

- **串行流程**：节点顺序执行
- **并行流程**：多个节点同时执行
- **分支流程**：根据条件选择路径

### 5.4 任务管理

- 待办任务列表
- 任务审批（通过/拒绝）
- 任务通知

### 5.5 执行日志追溯

完整记录每条流程执行：
- 节点执行日志
- 任务处理日志
- 错误日志

### 5.6 接口清单

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/workflow/definitions` | POST/GET | 工作流定义CRUD |
| `/api/workflow/trigger` | POST | 触发工作流 |
| `/api/workflow/tasks/pending` | GET | 获取待办任务 |
| `/api/workflow/tasks/{id}/complete` | POST | 完成任务 |
| `/api/workflow/instances/{id}/logs` | GET | 获取执行日志 |

## 8. 部署与配置

### 8.1 启动命令

```bash
# 初始化数据库
psql -d finrag4j -f sql/enterprise_schema.sql

# 启动服务
cd backend/finrag4j-java
mvn spring-boot:run
```

### 8.2 配置说明

**application.yml关键配置**：

```yaml
# 审计日志配置
audit:
  retention-years: 5  # 日志保留年限

# 多租户配置（商业版）
tenant:
  enabled: true       # 是否启用多租户
```

### 8.3 接口文档

启动后访问：http://localhost:8080/doc.html
