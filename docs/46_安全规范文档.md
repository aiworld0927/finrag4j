# 46_安全规范文档

## 1. 认证授权

### 1.1 JWT令牌管理

- Token有效期设置合理（建议24小时）
- 支持Token刷新机制
- 退出登录时Token无效化
- Token存储在HttpOnly Cookie中

### 1.2 权限控制

- 基于RBAC的权限模型
- 接口级别权限校验
- 文档级别权限控制
- 最小权限原则

## 2. 数据安全

### 2.1 数据加密

| 数据类型 | 加密方式 |
| :--- | :--- |
| 密码 | BCrypt/SCrypt哈希 |
| 敏感配置 | AES-256加密 |
| 传输数据 | HTTPS/TLS |

### 2.2 输入验证

- 参数校验使用Bean Validation（Java）或Pydantic（Python）
- 防止SQL注入：使用预编译语句
- 防止XSS攻击：输入输出转义
- 防止CSRF攻击：使用Token验证

### 2.3 数据脱敏

- 手机号：138****8888
- 身份证号：110****1234
- 邮箱：a***@xxx.com
- 银行卡号：**** **** **** 1234

## 3. 接口安全

### 3.1 接口限流

- 单用户限流：100次/分钟
- 单IP限流：1000次/分钟
- 使用Redis实现分布式限流

### 3.2 请求频率控制

- 登录接口：5次/分钟
- 短信验证码：1次/60秒
- 敏感操作：需要验证码

### 3.3 接口鉴权

- 所有接口需要Token验证
- 公开接口需要签名验证
- 内部接口限制访问IP

## 4. 日志安全

- 不记录密码等敏感信息
- 日志文件权限：仅管理员可读
- 日志传输加密
- 日志定期清理

## 5. 运维安全

### 5.1 服务器安全

- 关闭不必要的端口
- 使用防火墙限制访问
- 定期更新系统补丁
- 使用SSH密钥登录

### 5.2 数据库安全

- 数据库端口不对外暴露
- 使用强密码
- 限制数据库用户权限
- 定期备份数据

### 5.3 第三方依赖安全

- 定期检查依赖漏洞
- 使用安全的依赖版本
- 避免使用废弃的库
