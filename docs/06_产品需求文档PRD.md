# FinRag4j 产品需求文档 (PRD)

## 文档概述

本文档详细描述FinRag4j金融大模型RAG应用框架的产品需求，涵盖开源版和商业版全部功能。

## 1. 产品概述

### 1.1 产品定位

FinRag4j是面向金融行业的企业级大模型RAG应用框架，提供知识库管理、智能问答、合规自查、工作流编排等核心能力。

### 1.2 目标用户

| 用户类型 | 描述 | 主要需求 |
|---------|------|---------|
| 金融机构IT部门 | 银行、证券、保险IT团队 | 私有化部署、数据安全、合规审计 |
| 合规部门 | 金融合规管理人员 | 监管自查、风险识别、整改报告 |
| 客户经理 | 一线业务人员 | 产品推荐、营销方案、客户服务 |
| IT运维人员 | 系统运维工程师 | 监控告警、日志审计、性能优化 |

### 1.3 核心价值

- **私有化部署**：数据不出域，满足金融合规要求
- **金融垂直**：预置金融知识库、合规检查、信贷抽取
- **开箱即用**：完整前后端、一键部署、快速落地
- **灵活扩展**：工作流编排、Agent定制、模型路由

## 3. 非功能需求

### 3.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 问答响应时间 | <3s | P95延迟 |
| 并发用户数 | 100+ | 单机部署 |
| 文档解析速度 | 10页/秒 | PDF解析 |
| 向量检索延迟 | <100ms | Top10召回 |

### 3.2 安全要求

- 数据加密：传输层TLS加密
- 密码存储：BCrypt加密
- SQL注入：参数化查询
- XSS防护：输入过滤
- 权限控制：RBAC细粒度权限

### 3.3 合规要求

- 审计日志：保留5年
- 数据隔离：租户物理隔离
- 敏感词过滤：金融敏感词库
- 幻觉拦截：相似度阈值、来源验证

### 3.4 可用性要求

- 服务可用性：99.9%
- 故障恢复：<5分钟
- 数据备份：每日增量、每周全量

## 4. 技术架构

### 4.1 技术栈

| 层级 | 技术选型 |
|------|---------|
| 前端 | Vue3 + Vite + Element Plus |
| 后端 | Spring Boot 3.2 + MyBatis-Plus |
| 数据库 | PostgreSQL 16 + PGVector |
| 缓存 | Redis 7 |
| 消息队列 | RocketMQ 5.0 |
| 对象存储 | MinIO |
| 大模型 | Ollama / vLLM |
| Python服务 | FastAPI + uvicorn |
