# 45_代码风格指南

## 1. Java代码规范

### 1.1 命名规范

| 类型 | 规则 | 示例 |
| :--- | :--- | :--- |
| 类名 | PascalCase | UserService |
| 方法名 | camelCase | getUserById |
| 变量名 | camelCase | userName |
| 常量名 | UPPER_CASE_WITH_UNDERSCORE | MAX_RETRY_COUNT |
| 包名 | lowercase | com.finrag4j.auth |

### 1.2 代码格式

- 使用4个空格缩进
- 每行不超过120字符
- 左大括号不换行，右大括号单独一行
- 空行分隔逻辑块

### 1.3 注释规范

- Javadoc注释：类、公共方法必须有
- 单行注释：`// 注释内容`
- 多行注释：`/* 注释内容 */`
- TODO注释：`// TODO: 需要实现`

### 1.4 异常处理

```java
// 正确：捕获具体异常，提供错误信息
try {
    // 业务逻辑
} catch (SQLException e) {
    log.error("数据库操作失败: {}", e.getMessage(), e);
    throw new BusinessException("数据库操作失败");
}

// 错误：捕获Exception或Throwable
try {
    // 业务逻辑
} catch (Exception e) {
    // 过于宽泛的异常捕获
}
```

## 2. Python代码规范

### 2.1 命名规范

| 类型 | 规则 | 示例 |
| :--- | :--- | :--- |
| 类名 | PascalCase | DocumentParser |
| 函数名 | snake_case | parse_pdf_document |
| 变量名 | snake_case | file_path |
| 常量名 | UPPER_CASE_WITH_UNDERSCORE | MAX_FILE_SIZE |

### 2.2 代码格式

- 使用4个空格缩进
- 每行不超过120字符
- 空行分隔函数和类
- 函数和类之间空两行

### 2.3 注释规范

- 模块级注释：说明模块功能
- 函数注释：使用docstring格式
- 行内注释：`# 注释内容`

### 2.4 类型提示

```python
def process_document(file_path: str) -> dict:
    """处理文档
    
    Args:
        file_path: 文档路径
        
    Returns:
        处理结果字典
    """
    pass
```

## 3. 通用规范

### 3.1 日志规范

- 使用SLF4J（Java）或logging模块（Python）
- 日志级别：DEBUG、INFO、WARN、ERROR
- 避免在循环中打印日志
- 敏感信息不打印

### 3.2 错误处理

- 错误信息清晰明了
- 保留错误堆栈便于排查
- 提供错误码便于定位

### 3.3 代码复用

- 重复代码抽取为公共方法/函数
- 通用逻辑封装为工具类/模块
- 避免魔法数字，使用常量
