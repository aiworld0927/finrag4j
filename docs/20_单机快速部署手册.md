# FinRag4j 单机快速部署手册

## 文档概述

本文档提供FinRag4j单机环境的快速部署指南，适用于开发测试和POC演示场景。

**文档版本**: v1.0.0  
**适用环境**: 单机部署

---

## 1. 环境要求

### 1.1 硬件要求

| 配置项 | 最低要求 | 推荐配置 |
|--------|---------|---------|
| CPU | 4核 | 8核+ |
| 内存 | 16GB | 32GB+ |
| 硬盘 | 100GB SSD | 500GB SSD |
| GPU | 可选 | NVIDIA RTX 3090+ |

### 1.2 软件要求

| 软件 | 版本要求 |
|------|---------|
| Docker | 24.0+ |
| Docker Compose | 2.20+ |
| Git | 2.30+ |

---

## 2. 快速部署步骤

### 2.1 克隆项目

```bash
git clone https://github.com/finrag4j/finrag4j.git
cd finrag4j
```

### 2.2 配置环境变量

```bash
cp .env.example .env
# 编辑.env文件，配置数据库密码等
```

### 2.3 一键启动

```bash
cd deploy
docker-compose up -d
```

### 2.4 检查服务状态

```bash
docker-compose ps
```

### 2.5 访问服务

| 服务 | 地址 |
|------|------|
| 前端界面 | http://localhost |
| API文档 | http://localhost:8080/doc.html |
| MinIO控制台 | http://localhost:9001 |

---

## 3. 初始化配置

### 3.1 创建知识库

```bash
curl -X POST "http://localhost:8080/api/kb?tenantId=1" \
  -H "Content-Type: application/json" \
  -d '{
    "kbName": "金融法规知识库",
    "kbCode": "finance_regulation",
    "description": "金融监管法规知识库"
  }'
```

### 3.2 上传测试文档

通过前端界面上传测试文档，或使用API：

```bash
curl -X POST "http://localhost:8080/api/documents/upload?tenantId=1" \
  -F "file=@test.pdf"
```

### 3.3 测试问答

```bash
curl -X POST "http://localhost:8080/api/chat/new" \
  -d "question=测试问题&kbId=1&tenantId=1&userId=1"
```

## 4. 常见问题

### Q1: 服务启动失败

检查端口占用：
```bash
netstat -tlnp | grep -E "80|8080|5432|6379|9000"
```

### Q2: 数据库连接失败

检查PostgreSQL状态：
```bash
docker-compose logs postgres
```

### Q3: 模型加载失败

检查Ollama服务：
```bash
docker-compose logs ollama
```

---

## 5. 停止服务

```bash
docker-compose down
```

保留数据：
```bash
docker-compose down
# 数据卷会保留
```

清理数据：
```bash
docker-compose down -v
```
