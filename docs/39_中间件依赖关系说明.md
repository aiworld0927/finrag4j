## 2. 详细依赖关系

### 2.1 基础中间件（无依赖）

| 服务 | 端口 | 依赖 | 说明 |
|------|------|------|------|
| PostgreSQL | 5432 | 无 | 主数据库，存储业务数据和 Nacos 配置 |
| Redis | 6379 | 无 | 缓存、会话、限流 |
| MinIO | 9000/9001 | 无 | 对象存储，存储文档文件 |
| RocketMQ NameServer | 9876 | 无 | 消息队列注册中心 |
| Nacos | 8848/8080 | PostgreSQL | **8848** 用于应用客户端连接，**8080** 用于 Web 控制台访问 |

### 2.2 依赖型中间件

| 服务 | 端口 | 依赖 | 说明 |
|------|------|------|------|
| Nacos | 8848 | PostgreSQL | 配置中心，依赖数据库存储配置 |
| RocketMQ Broker | 10911 | NameServer | 消息代理，依赖 NameServer |
| RocketMQ Dashboard | 8088 | NameServer | 管理控制台 |

### 2.3 业务服务

| 服务 | 端口 | 依赖 | 说明 |
|------|------|------|------|
| Python Service | 8001 | Nacos(可选) | 文档预处理服务 |
| Gateway | 8080 | Nacos, Redis | API 网关 |
| Auth | 8081 | Nacos, PostgreSQL, Redis | 认证服务 |
| Document | 9082 | Nacos, PostgreSQL, Redis, MinIO, Python | 文档服务 |
| Search | 8083 | Nacos, PostgreSQL, Redis | 检索服务 |
| Agent | 8084 | Nacos, PostgreSQL, Redis | 智能代理服务 |
| Frontend | 80 | Gateway | 前端服务 |

## 3. 启动顺序

### 3.1 推荐启动顺序

```
第1层: 基础服务（无依赖）
├── PostgreSQL
├── Redis
├── MinIO
└── RocketMQ NameServer

第2层: 依赖型中间件
├── Nacos (等待 PostgreSQL 健康检查通过)
└── RocketMQ Broker (等待 NameServer 健康检查通过)

第3层: 辅助服务
└── RocketMQ Dashboard

第4层: 核心服务
└── Python Service (等待 Nacos 健康检查通过)

第5层: 业务微服务
├── Gateway (等待 Nacos, Redis)
├── Auth (等待 Nacos, PostgreSQL, Redis)
├── Search (等待 Nacos, PostgreSQL, Redis)
└── Agent (等待 Nacos, PostgreSQL, Redis)

第6层: 业务服务
└── Document (等待 Nacos, PostgreSQL, Redis, MinIO, Python)

第7层: 前端
└── Frontend (等待 Gateway)
```

### 3.2 启动时间估算

| 层级 | 服务 | 等待时间 | 累计时间 |
|------|------|----------|----------|
| 第1层 | 基础服务 | 30-60秒 | 60秒 |
| 第2层 | Nacos/Broker | 60-90秒 | 150秒 |
| 第3层 | Dashboard | 10秒 | 160秒 |
| 第4层 | Python | 30秒 | 190秒 |
| 第5层 | 微服务 | 60秒 | 250秒 |
| 第6层 | Document | 30秒 | 280秒 |
| 第7层 | Frontend | 10秒 | 290秒 |

**总启动时间：约 5 分钟**

## 4. Docker Compose 配置要点

### 4.1 depends_on 配置

```yaml
# Nacos 依赖 PostgreSQL
nacos:
  depends_on:
    postgres:
      condition: service_healthy

# RocketMQ Broker 依赖 NameServer
rocketmq-broker:
  depends_on:
    rocketmq-namesrv:
      condition: service_healthy

# 业务服务依赖中间件
auth-service:
  depends_on:
    nacos:
      condition: service_healthy
    postgres:
      condition: service_healthy
    redis:
      condition: service_healthy
```

### 4.2 健康检查配置

```yaml
# PostgreSQL 健康检查
healthcheck:
  test: ["CMD-SHELL", "pg_isready -U postgres"]
  interval: 10s
  timeout: 5s
  retries: 5

# Redis 健康检查
healthcheck:
  test: ["CMD", "redis-cli", "ping"]
  interval: 10s
  timeout: 5s
  retries: 5

# Nacos 健康检查
healthcheck:
  test: ["CMD", "curl", "-f", "http://localhost:8848/nacos/v1/console/health/readiness"]
  interval: 15s
  timeout: 10s
  retries: 10
  start_period: 60s
```

## 5. K8s 配置要点

### 5.1 Init Containers

```yaml
# 等待依赖服务就绪
initContainers:
- name: wait-for-postgres
  image: busybox:1.32
  command: ['sh', '-c', 'until nc -z postgres 5432; do echo waiting for postgres; sleep 2; done;']
- name: wait-for-redis
  image: busybox:1.32
  command: ['sh', '-c', 'until nc -z redis 6379; do echo waiting for redis; sleep 2; done;']
- name: wait-for-nacos
  image: busybox:1.32
  command: ['sh', '-c', 'until nc -z nacos 8848; do echo waiting for nacos; sleep 2; done;']
```

### 5.2 Readiness Probe

```yaml
readinessProbe:
  httpGet:
    path: /actuator/health
    port: 8080
  initialDelaySeconds: 30
  periodSeconds: 5
```

## 6. 服务状态检查

### 6.1 检查命令

```bash
# 检查所有中间件状态
docker-compose -f docker-compose-base.yml ps

# 检查单个服务日志
docker logs -f finrag4j-nacos

# 检查服务健康状态
curl http://localhost:8848/nacos/v1/console/health/readiness
curl http://localhost:8001/health
curl http://localhost:8080/actuator/health
```

### 6.2 端口检查

```bash
# Linux
netstat -tlnp | grep -E '5432|6379|8848|9000|9876|8001|8080'

# Windows
netstat -ano | findstr "5432 6379 8848 9000 9876 8001 8080"
```

## 7. 故障排查

### 7.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Nacos 启动失败 | PostgreSQL 未就绪 | 检查 PostgreSQL 健康状态 |
| Broker 启动失败 | NameServer 未就绪 | 检查 NameServer 日志 |
| 微服务无法注册 | Nacos 未就绪 | 等待 Nacos 健康检查通过 |
| 文档上传失败 | MinIO 未就绪 | 检查 MinIO 健康状态 |

### 7.2 重启策略

```bash
# 完全重启（保留数据）
docker-compose down
docker-compose up -d

# 完全重启（清除数据）
rm -rf ./data/*
docker-compose up -d
```
