## 2. 代码规范

### 2.1 Java代码规范

**遵循标准：** 阿里巴巴Java开发手册

**命名规范：**
```java
// 类名：大驼峰
public class UserService { }

// 方法名、变量名：小驼峰
private String userName;
public void getUserById() { }

// 常量：全大写下划线分隔
public static final String DEFAULT_PASSWORD = "admin";

// 包名：全小写
package com.finrag4j.auth.controller;
```

**注释规范：**
```java
/**

public class UserService {

     * 根据用户名获取用户信息
     *
     * @param username 用户名
     * @return 用户信息
     */
    public User getUserByUsername(String username) {
        // TODO: 实现逻辑
    }
}
```

## 3. 开发流程

### 3.1 Git工作流

```
main        ──────────────────────────────────→ 生产版本
             ↖                               ↗
              ↖      ↗                       ↗
develop    ────────→───────────────────────────→ 开发版本
              ↖   ↗ ↖   ↗ ↖   ↗ ↖   ↗
feature/xxx ──→───→──→──→──→──→──→──→──→──→──
              ↖   ↗ ↖   ↗ ↖   ↗ ↖   ↗
bugfix/xxx ──→───→──→──→──→──→──→──→──→──→──→
```

**分支说明：**
| 分支 | 说明 | 命名规则 |
|------|------|----------|
| main | 生产分支 | - |
| develop | 开发分支 | - |
| feature | 功能分支 | feature/功能名称 |
| bugfix | 修复分支 | bugfix/问题描述 |
| hotfix | 热修复分支 | hotfix/问题描述 |

### 3.2 代码提交规范

**提交格式：**
```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type类型：**
| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | 修复bug |
| docs | 文档更新 |
| style | 代码格式（不影响功能） |
| refactor | 重构（不是新功能或修复bug） |
| perf | 性能优化 |
| test | 测试相关 |
| chore | 构建或辅助工具 |

**提交示例：**
```bash
# 好示例
git commit -m "feat(auth): 添加用户登录接口"
git commit -m "fix(document): 修复文档上传失败问题"
git commit -m "docs: 更新部署手册"

# 不好示例
git commit -m "update"
git commit -m "fix bug"
```

### 3.3 代码审查

提交PR前请确保：
- [ ] 代码通过所有单元测试
- [ ] 代码格式符合规范
- [ ] 添加必要的注释
- [ ] 更新相关文档

## 4. 本地运行指南

### 4.1 环境变量配置

#### Windows PowerShell

```powershell
# 在 $PROFILE 中添加以下内容（永久生效）
notepad $PROFILE

# 添加以下函数
function Set-Finrag4jEnv {
    $env:NACOS_HOST = "11.0.1.190"
    $env:NACOS_PORT = "8848"
    $env:NACOS_USERNAME = "nacos"
    $env:NACOS_PASSWORD = "nacos"
    $env:POSTGRES_HOST = "11.0.1.190"
    $env:POSTGRES_PORT = "5432"
    $env:POSTGRES_DB = "nacos"
    $env:POSTGRES_USER = "nacos"
    $env:POSTGRES_PASSWORD = "nacos"
    $env:REDIS_HOST = "11.0.1.190"
    $env:REDIS_PORT = "6379"
    $env:MINIO_HOST = "11.0.1.190"
    $env:MINIO_PORT = "9000"
    $env:MINIO_ROOT_USER = "minioadmin"
    $env:MINIO_ROOT_PASSWORD = "minioadmin"
    Write-Host "FinRag4j环境变量已配置" -ForegroundColor Green
}

# 使用
Set-Finrag4jEnv
```

### 4.4 Python服务本地启动

#### 4.4.1 系统依赖（必装）

Python服务依赖OpenCV，需要先安装系统库：

**Ubuntu 24.04:**
```bash
sudo apt update && sudo apt install -y libgl1 libglib2.0-0 libsm6 libxext6 libxrender-dev
```

**Ubuntu 20.04/22.04:**
```bash
sudo apt update && sudo apt install -y libgl1-mesa-glx libglib2.0-0 libsm6 libxext6 libxrender-dev
```

**Windows/Mac:** 无需额外安装

#### 4.4.2 模型依赖

Python服务使用PaddleOCR，首次启动会自动下载模型。模型存放路径可通过环境变量配置：

```bash
# 指定模型存储目录（推荐）
export PADDLEX_HOME=/data/models

# 优先使用modelscope源下载
export PADDLE_PDX_MODEL_SOURCE=modelscope
```

#### 4.4.3 启动服务

```bash
cd backend/finrag4j-python

# 安装依赖
uv sync

# 开发模式运行
uv run python main.py

# 或激活虚拟环境
.venv\Scripts\activate  # Windows
source .venv/bin/activate  # Linux/Mac
python main.py
```

**注意：** 首次启动会自动下载PP-OCRv6等模型，请确保网络连接。

### 5.2 API调用

```javascript
// src/api/index.js
import axios from 'axios'

const api = axios.create({
  baseURL: '/api',
  timeout: 30000
})

// 请求拦截器
api.interceptors.request.use(
  config => {
    const token = localStorage.getItem('token')
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }
    return config
  },
  error => Promise.reject(error)
)

export default api
```

### 5.3 路由配置

```javascript
// src/router/index.js
import { createRouter, createWebHistory } from 'vue-router'
import store from '@/stores/auth'

const routes = [
  {
    path: '/login',
    name: 'Login',
    component: () => import('@/views/Login.vue'),
    meta: { requiresAuth: false }
  },
  {
    path: '/',
    name: 'Home',
    component: () => import('@/views/Home.vue'),
    meta: { requiresAuth: true }
  }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

router.beforeEach((to, from, next) => {
  if (to.meta.requiresAuth && !store.state.token) {
    next('/login')
  } else {
    next()
  }
})

export default router
```

## 7. 常见问题

### 7.1 端口占用

**问题：** Address already in use

**解决：**
```bash
# Windows
netstat -ano | findstr 8085
taskkill /PID <PID> /F

# Linux
lsof -i :8085
kill -9 <PID>
```

### 7.2 依赖下载失败

**问题：** Maven/Gradle依赖下载超时

**解决：**
```bash
# 配置阿里云镜像
mvn settings.xml 添加：
<mirror>
  <id>aliyun</id>
  <mirrorOf>central</mirrorOf>
  <url>https://maven.aliyun.com/repository/public</url>
</mirror>
```

### 7.3 数据库连接失败

**问题：** Unable to connect to database

**解决：**
1. 检查PostgreSQL服务是否运行
2. 验证用户名密码是否正确
3. 检查防火墙设置

### 7.4 前端代理不生效

**问题：** 请求无法代理到后端

**解决：**
1. 重启前端开发服务器
2. 检查 vite.config.js 代理配置
3. 确认网关服务已启动
