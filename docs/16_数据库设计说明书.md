# FinRag4j 数据库设计说明书

## 文档信息
- 版本：v1.0
- 更新日期：2026-06-25
- 适用版本：FinRag4j v1.0+

---

## 1. 数据库架构概览

### 1.1 数据库划分

| 数据库名 | 说明 | 服务模块 |
|---------|------|---------|
| finrag4j_auth | 认证授权数据 | finrag4j-auth |
| finrag4j_document | 文档与知识库数据 | finrag4j-document |
| finrag4j_search | 向量检索数据 | finrag4j-search |
| finrag4j_agent | Agent与对话数据 | finrag4j-agent |

### 1.2 技术选型

- **数据库**：PostgreSQL 15+
- **扩展**：PGVector（向量存储）
- **ORM框架**：MyBatis-Plus 3.5.5
- **连接池**：Druid 1.2.20
- **分库策略**：按业务模块垂直分库
- **多租户**：tenant_id 字段隔离

## 6. 数据库设计规范

### 6.1 命名规范

- **表名**：小写下划线分隔，如 `sys_user`
- **字段名**：小写下划线分隔，如 `create_time`
- **索引名**：`idx_表名_字段名`
- **外键**：业务层控制，不设置数据库外键约束

### 6.2 通用字段

所有业务表必须包含以下字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 主键，Snowflake算法 |
| tenant_id | BIGINT | 租户ID（多租户隔离） |
| create_time | TIMESTAMP | 创建时间 |
| update_time | TIMESTAMP | 更新时间 |
| deleted | INTEGER | 逻辑删除（MyBatis-Plus） |

### 6.3 数据类型规范

| 场景 | 推荐类型 | 说明 |
|------|---------|------|
| 主键 | BIGINT | Snowflake ID |
| 金额 | DECIMAL(19,4) | 金融精度要求 |
| 状态枚举 | VARCHAR(20) | 便于扩展 |
| JSON数据 | JSON | PostgreSQL原生支持 |
| 大文本 | TEXT | 无长度限制 |
| 向量 | vector(1536) | PGVector扩展 |

### 6.4 索引设计原则

1. **主键索引**：所有表必须有主键
2. **查询索引**：WHERE条件高频字段加索引
3. **联合索引**：遵循最左前缀原则
4. **向量索引**：ivfflat算法，lists=100
5. **避免过度索引**：单表索引不超过5个
