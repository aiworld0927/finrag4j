## 2. 测试覆盖率要求

| 层级 | 覆盖率要求 | 说明 |
|------|-----------|------|
| 核心服务（auth/document/search） | 行覆盖率 >= 80% | 核心业务逻辑 |
| 工具类/公共模块 | 行覆盖率 >= 90% | 复用性高，影响面广 |
| Controller/API层 | 行覆盖率 >= 70% | 接口参数校验 |
| Python预处理服务 | 行覆盖率 >= 75% | 文档解析等核心流程 |
| 前端组件 | 行覆盖率 >= 60% | 关键业务组件 |

**分支覆盖率要求：** 不低于行覆盖率的 70%

## 4. 测试结构规范（AAA模式）

```java
@Test
@DisplayName("登录成功：返回有效的JWT Token")
void testLoginSuccess() {
    // Arrange - 准备测试数据
    LoginRequest request = new LoginRequest();
    request.setUsername("admin");
    request.setPassword("admin");
    
    when(userMapper.selectByUsername("admin"))
        .thenReturn(mockUser);
    
    // Act - 执行被测方法
    LoginResult result = authService.login(request);
    
    // Assert - 验证结果
    assertThat(result.getCode()).isEqualTo(200);
    assertThat(result.getData()).isNotNull();
    assertThat(result.getData().getAccessToken()).isNotBlank();
}
```

## 5. 测试分类

### 5.1 单元测试（Unit Test）

**范围：** 单个类/方法的逻辑

**要求：**
- 不依赖外部资源（数据库、网络、文件系统）
- 使用 Mockito 模拟依赖
- 测试执行时间 < 100ms

```java
@ExtendWith(MockitoExtension.class)
class UserServiceTest {
    @Mock
    private UserMapper userMapper;
    
    @InjectMocks
    private UserService userService;
    
    @Test
    void testGetUserById() {
        // 纯内存测试，无外部依赖
    }
}
```

### 5.2 集成测试（Integration Test）

**范围：** 多组件协作 + 真实中间件

**要求：**
- 使用 Testcontainers 启动真实数据库
- 测试事务回滚，不污染数据

```java
@SpringBootTest
@Testcontainers
class AuthIntegrationTest {
    @Container
    static PostgreSQLContainer<?> postgres = 
        new PostgreSQLContainer<>("postgres:15");
    
    @Test
    void testUserRegistration() {
        // 使用真实数据库测试
    }
}
```

### 5.3 API 测试

**范围：** HTTP 接口端到端测试

```java
@AutoConfigureMockMvc
@SpringBootTest
class AuthControllerTest {
    @Autowired
    private MockMvc mockMvc;
    
    @Test
    void testLoginEndpoint() throws Exception {
        mockMvc.perform(post("/auth/login")
                .contentType(MediaType.APPLICATION_JSON)
                .content("{\"username\":\"admin\",\"password\":\"admin\"}"))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.code").value(200));
    }
}
```

## 7. 执行命令

### 7.1 Java 后端

```bash
# 运行所有测试
cd backend
mvn test

# 运行单个测试类
mvn test -Dtest=UserServiceTest

# 运行单个测试方法
mvn test -Dtest=UserServiceTest#testLoginSuccess

# 生成覆盖率报告
mvn jacoco:report
# 报告位置：target/site/jacoco/index.html
```

### 7.2 Python 服务

```bash
cd backend/finrag4j-python

# 运行所有测试
pytest

# 运行指定模块
pytest tests/test_parser/

# 生成覆盖率报告
pytest --cov=src --cov-report=html
```

### 7.3 前端

```bash
cd frontend

# 运行测试
npm run test

# 生成覆盖率报告
npm run test:coverage
```

## 9. 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 测试偶发失败 | 数据未清理/并发冲突 | 使用 @Transactional 或手动清理 |
| 覆盖率不达标 | 缺少异常分支测试 | 补充边界条件、异常场景测试 |
| 测试执行慢 | 启动了 Spring 上下文 | 使用 @WebMvcTest/@DataJpaTest 精简 |
| 时区问题 | 数据库与JVM时区不一致 | 统一设置 UTC 时区 |
