## 0. 环境配置脚本（Linux开发环境）

### 0.1 脚本位置

```
scripts/setup-env.sh
```

### 0.2 支持的环境

| 环境名称 | 中间件地址 | Nacos端口 | 说明 |
|----------|------------|------------|------|
| devhome | 11.0.1.190 | 8848 | 家庭/主开发环境 |
| devcompany | 192.168.222.186 | 8080 | 公司开发环境 |

### 0.3 使用方式

```bash
# 进入脚本目录
cd /coding/finrag4j/scripts

# 加载环境变量（选择对应环境）
source setup-env.sh devhome      # 家庭环境
source setup-env.sh devcompany   # 公司环境

# 验证配置
echo $NACOS_HOST
echo $POSTGRES_HOST
echo $MIDDLEWARE_HOST

# 检查Java和Maven
java -version
mvn -v
```

## 2. 开发环境配置（Windows + Linux中间件）

### 2.1 前置条件

**Windows 开发机要求：**
- JDK 17+
- Maven 3.8+
- Python 3.12+
- Node.js 18+
- uv（Python包管理器）

**Linux 中间件服务器要求：**
- Docker & Docker Compose
- 已部署：Nacos、PostgreSQL、Redis、MinIO、RocketMQ

### 2.2 中间件部署（Linux服务器）

在Linux服务器上执行：

```bash
# 1. 进入deploy目录
cd /path/to/finrag4j/deploy

# 2. 复制环境配置
cp .env.dev1 .env

# 3. 启动基础中间件
docker-compose -f docker-compose-base.yml up -d

# 4. 检查服务状态
docker-compose -f docker-compose-base.yml ps
```

**验证中间件服务：**

```bash
# Nacos 控制台
http://11.0.1.190:8848/nacos
用户名/密码: nacos/nacos

# MinIO 控制台
http://11.0.1.190:9001
用户名/密码: minioadmin/minioadmin

# RocketMQ Dashboard
http://11.0.1.190:8088
```

### 2.4 启动服务

#### 2.4.1 启动 Python 服务

```powershell
# 进入Python模块目录
cd backend\finrag4j-python

# 方式一：禁用Nacos（本地调试）
$env:NACOS_ENABLED="false"
$env:DEBUG_MODE="true"
uv run python main.py

# 方式二：启用Nacos（连接远程配置中心）
$env:NACOS_ENABLED="true"
$env:NACOS_HOST="11.0.1.190"
uv run python main.py
```

**验证：**
```powershell
curl http://localhost:8001/health
```

**Maven 方式：**


## 3. 多开发环境切换

### 3.1 环境切换脚本

创建切换脚本 `switch-env.ps1`：

```powershell
param(
    [Parameter(Mandatory=$true)]
    [ValidateSet("dev1", "dev2", "test", "prod")]
    [string]$env
)

$envMap = @{
    "dev1" = "11.0.1.190"
    "dev2" = "192.168.222.186"
    "test" = "测试环境IP"
    "prod" = "生产环境IP"
}

$ip = $envMap[$env]

$env:NACOS_HOST = $ip
$env:POSTGRES_HOST = $ip
$env:REDIS_HOST = $ip
$env:MINIO_HOST = $ip
$env:ROCKETMQ_NAMESRV_HOST = $ip

Write-Host "已切换到 $env 环境，中间件IP: $ip" -ForegroundColor Green
```

**使用方式：**

```powershell
# 切换到开发环境1
.\switch-env.ps1 -env dev1

# 切换到开发环境2
.\switch-env.ps1 -env dev2
```

### 3.2 快速切换命令

```powershell
# 开发环境1
$env:NACOS_HOST="11.0.1.190"; $env:POSTGRES_HOST="11.0.1.190"; $env:REDIS_HOST="11.0.1.190"

# 开发环境2
$env:NACOS_HOST="192.168.222.186"; $env:POSTGRES_HOST="192.168.222.186"; $env:REDIS_HOST="192.168.222.186"
```

## 6. 常见问题排查

### 6.1 连接中间件失败

**症状：**
```
Connection refused: NACOS_HOST:8848
```

**排查步骤：**

```bash
# 1. 检查中间件服务状态
docker ps | grep nacos

# 2. 检查端口是否开放
telnet 11.0.1.190 8848

# 3. 检查防火墙
sudo firewall-cmd --list-ports
sudo ufw status

# 4. 检查Docker网络
docker network ls
docker network inspect finrag4j-network
```

**解决方案：**

```bash
# 开放端口
sudo firewall-cmd --add-port=8848/tcp --permanent
sudo firewall-cmd --reload

# 或使用 ufw
sudo ufw allow 8848/tcp
```

### 6.2 Nacos连接超时

**症状：**
```
Nacos connection timeout
```

**解决方案：**

```powershell
# Windows端检查网络
ping 11.0.1.190
Test-NetConnection 11.0.1.190 -Port 8848

# 检查Nacos健康状态
curl http://11.0.1.190:8848/nacos/v1/console/health/readiness
```

### 6.3 数据库连接失败

**症状：**
```
PSQLException: Connection to 11.0.1.190:5432 refused
```

**解决方案：**

```bash
# 检查PostgreSQL状态
docker logs finrag4j-postgres

# 检查pg_hba.conf配置
docker exec -it finrag4j-postgres cat /var/lib/postgresql/data/pg_hba.conf

# 添加远程访问权限
# pg_hba.conf 添加:
host all all 0.0.0.0/0 md5
```

## 8. 快速启动清单

### 8.1 开发环境启动顺序

```
1. 确认Linux中间件服务运行
2. 配置Windows环境变量
3. 启动Python服务 (8001)
4. 启动Gateway (8080)
5. 启动Auth (8081)
6. 启动Document (8082)
7. 启动Search (8083)
8. 启动Agent (8084)
9. 启动Frontend (80)
```

### 8.2 一键启动脚本

创建 `start-dev.ps1`：

```powershell
# 设置环境变量
$env:NACOS_HOST="11.0.1.190"
$env:POSTGRES_HOST="11.0.1.190"
$env:REDIS_HOST="11.0.1.190"
$env:MINIO_ENDPOINT="http://11.0.1.190:9000"
$env:NACOS_ENABLED="false"
$env:DEBUG_MODE="true"

# 启动Python服务
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd backend\finrag4j-python; uv run python main.py"

# 等待Python启动
Start-Sleep -Seconds 5

# 启动Java服务
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd backend\finrag4j-gateway; mvn spring-boot:run"
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd backend\finrag4j-auth; mvn spring-boot:run"
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd backend\finrag4j-document; mvn spring-boot:run"
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd backend\finrag4j-search; mvn spring-boot:run"
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd backend\finrag4j-agent; mvn spring-boot:run"

# 启动前端
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd frontend; npm run dev"

Write-Host "所有服务启动完成！" -ForegroundColor Green
```
