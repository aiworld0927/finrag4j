## 2. Docker Compose部署

### 2.1 目录结构

```
deploy/
├── docker-compose.yml          # 主配置文件
├── docker-compose-base.yml     # 基础中间件
├── docker-compose-gateway.yml  # 网关服务
├── docker-compose-python.yml   # Python服务
└── .env                        # 环境变量
```

### 2.2 配置环境变量

在 `deploy` 目录下创建或编辑 `.env` 文件：

```bash
# 中间件配置
NACOS_HOST=11.0.1.190
NACOS_PORT=8848
NACOS_USERNAME=nacos
NACOS_PASSWORD=nacos

POSTGRES_HOST=11.0.1.190
POSTGRES_PORT=5432
POSTGRES_DB=nacos
POSTGRES_USER=nacos
POSTGRES_PASSWORD=nacos

REDIS_HOST=11.0.1.190
REDIS_PORT=6379

MINIO_HOST=11.0.1.190
MINIO_PORT=9000
MINIO_ROOT_USER=minioadmin
MINIO_ROOT_PASSWORD=minioadmin

# 服务端口
GATEWAY_PORT=8085
AUTH_PORT=8081
DOCUMENT_PORT=8082
SEARCH_PORT=8083
AGENT_PORT=8086
PYTHON_SERVICE_PORT=8001
```

### 2.3 启动基础中间件

```bash
cd deploy

# 启动Nacos、PostgreSQL、Redis、MinIO、RocketMQ
docker-compose -f docker-compose-base.yml up -d

# 检查服务状态
docker-compose -f docker-compose-base.yml ps
```

### 2.4 启动Java微服务

```bash
# 启动网关
docker-compose -f docker-compose-gateway.yml up -d

# 启动认证服务
docker-compose up -d finrag4j-auth

# 启动文档服务
docker-compose up -d finrag4j-document

# 启动搜索服务
docker-compose up -d finrag4j-search

# 启动Agent服务
docker-compose up -d finrag4j-agent
```

### 2.5 启动Python服务

```bash
# 构建Python服务镜像
docker build -f ../backend/finrag4j-python/Dockerfile -t finrag4j-python:latest ../backend/finrag4j-python

# 启动Python服务
docker-compose -f docker-compose-python.yml up -d
```

### 2.6 启动前端

```bash
cd frontend

# 构建生产版本
npm run build

# 启动Nginx服务
docker run -d -p 80:80 -v ./dist:/usr/share/nginx/html nginx:alpine
```

## 3. 本地Java服务部署

### 3.1 环境变量配置

在启动Java服务前，需要配置以下环境变量：

```powershell
# PowerShell
$env:JAVA_HOME="C:\dev\Java\jdk-21.0.11"
$env:NACOS_HOST="11.0.1.190"
$env:NACOS_PORT="8848"
$env:NACOS_USERNAME="nacos"
$env:NACOS_PASSWORD="nacos"
$env:NACOS_NAMESPACE="public"
$env:POSTGRES_HOST="11.0.1.190"
$env:POSTGRES_PORT="5432"
$env:POSTGRES_DB="nacos"
$env:POSTGRES_USER="nacos"
$env:POSTGRES_PASSWORD="nacos"
$env:REDIS_HOST="11.0.1.190"
$env:REDIS_PORT="6379"
$env:MINIO_HOST="11.0.1.190"
$env:MINIO_PORT="9000"
$env:MINIO_ROOT_USER="minioadmin"
$env:MINIO_ROOT_PASSWORD="minioadmin"
```

### 3.2 使用启动脚本

项目提供了 `backend\run.bat` 启动脚本：

```batch
cd backend
run.bat
```

脚本会自动启动以下服务：
- FinRag4j-Gateway (8085)
- FinRag4j-Auth (8081)
- FinRag4j-Document (8082)
- FinRag4j-Search (8083)
- FinRag4j-Agent (8086)

### 3.3 Maven命令启动

```bash
cd backend

# 编译项目
mvn clean install -DskipTests

# 启动网关服务
cd finrag4j-gateway
mvn spring-boot:run

# 新开终端，启动认证服务
cd ../finrag4j-auth
mvn spring-boot:run

# 新开终端，启动文档服务
cd ../finrag4j-document
mvn spring-boot:run

# 新开终端，启动搜索服务
cd ../finrag4j-search
mvn spring-boot:run

# 新开终端，启动Agent服务
cd ../finrag4j-agent
mvn spring-boot:run
```

### 3.4 IDEA启动配置

在IDEA中创建运行配置：

```

环境变量：
  NACOS_HOST=11.0.1.190
  POSTGRES_HOST=11.0.1.190
  REDIS_HOST=11.0.1.190
  MINIO_HOST=11.0.1.190
```

### 3.5 JAR包直接启动

## 4. 本地Python服务部署

### 4.1 系统依赖（必装）

Python服务依赖OpenCV，需要先安装系统库：

**Ubuntu 24.04:**
```bash
sudo apt update && sudo apt install -y libgl1 libglib2.0-0 libsm6 libxext6 libxrender-dev
```

**Ubuntu 20.04/22.04:**
```bash
sudo apt update && sudo apt install -y libgl1-mesa-glx libglib2.0-0 libsm6 libxext6 libxrender-dev
```

**CentOS/RHEL:**
```bash
sudo yum install -y mesa-libGL glib2 libSM libXext libXrender
```

### 4.2 环境准备

```bash
# 安装uv包管理器
pip install uv

# 进入Python服务目录
cd backend/finrag4j-python

# 安装依赖
uv sync
```

### 4.3 环境变量配置

```powershell
# PowerShell (Windows)
$env:NACOS_HOST="11.0.1.190"
$env:NACOS_PORT="8848"
$env:NACOS_USERNAME="nacos"
$env:NACOS_PASSWORD="nacos"
$env:MINIO_HOST="11.0.1.190"
$env:MINIO_PORT="9000"
$env:MINIO_ROOT_USER="minioadmin"
$env:MINIO_ROOT_PASSWORD="minioadmin"
$env:PYTHON_SERVICE_PORT="8001"
```

```bash
# Linux/Mac

export MINIO_ROOT_PASSWORD="minioadmin"
```

### 4.4 模型依赖配置

Python服务使用PaddleOCR进行文档识别，首次启动会自动下载模型。

#### 4.4.1 模型下载清单

| 模型名称 | 存储路径 |
|---------|---------|
| PP-LCNet_x1_0_doc_ori | /root/.paddlex/official_models/PP-LCNet_x1_0_doc_ori |
| UVDoc | /root/.paddlex/official_models/UVDoc |
| PP-LCNet_x1_0_textline_ori | /root/.paddlex/official_models/PP-LCNet_x1_0_textline_ori |
| PP-OCRv6_medium_det | /root/.paddlex/official_models/PP-OCRv6_medium_det |
| PP-OCRv6_medium_rec | /root/.paddlex/official_models/PP-OCRv6_medium_rec |

#### 4.4.2 模型路径管理

**方案一：环境变量重定向（推荐）**

设置 `PADDLEX_HOME` 环境变量指定模型存放根目录：

```bash
# Linux/Mac
export PADDLEX_HOME=/data/models

# Windows PowerShell
$env:PADDLEX_HOME="D:\models"
```

模型会自动下载到 `$PADDLEX_HOME/official_models/` 目录下。

**方案二：代码中设置**

```python
import os
os.environ['PADDLEX_HOME'] = '/custom/models'
```

**方案三：软链接**

```bash
# 将默认目录链接到实际存储位置
ln -s /data/models /root/.paddlex
```

**方案四：模型源配置**

优先使用 modelscope 下载，减少网络问题：

```bash
export PADDLE_PDX_MODEL_SOURCE=modelscope
```

**方案五：离线部署**

若网络受限，可提前从 ModelScope 下载模型文件，设置：

```bash
export PADDLE_PDX_DISABLE_MODEL_SOURCE_CHECK=True
```

### 4.5 启动服务

```bash
cd backend/finrag4j-python

# 开发模式（自动重载）
uv run python main.py

# 或激活虚拟环境后运行
.venv\Scripts\activate  # Windows
source .venv/bin/activate  # Linux/Mac
python main.py
```

### 4.4 禁用Nacos本地调试

```powershell
# 禁用Nacos，使用本地配置
$env:NACOS_ENABLED="false"
$env:DEBUG_MODE="true"

# 启动服务
uv run python main.py
```

### 4.5 Docker启动Python服务

```bash
cd backend/finrag4j-python

# 构建镜像
docker build -t finrag4j-python:latest .

# 运行容器
docker run -d -p 8001:8001 \
  -e NACOS_HOST=11.0.1.190 \
  -e NACOS_PORT=8848 \
  -e MINIO_HOST=11.0.1.190 \
  finrag4j-python:latest
```


访问 http://localhost:5173

### 5.2 生产构建

```bash
cd frontend

# 构建生产版本
npm run build

# 输出目录: dist/
```

### 5.3 Nginx部署

```nginx
server {
    listen 80;
    server_name localhost;
    
    root /usr/share/nginx/html;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    location /api {
        proxy_pass http://11.0.1.190:8085;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 5.4 Docker部署前端

```bash
cd frontend

# 构建
npm run build

# 使用Nginx运行
docker run -d -p 80:80 \
  -v ./dist:/usr/share/nginx/html \
  nginx:alpine
```

## 8. 常见问题

### 8.1 数据库连接失败

**问题**：PSQLException: password authentication failed

**解决**：检查 `POSTGRES_USER` 和 `POSTGRES_PASSWORD` 是否与数据库配置一致

### 8.2 Nacos连接失败

**问题**：Unable to connect to Nacos

**解决**：
1. 检查Nacos服务是否运行
2. 验证网络连通性 `telnet 11.0.1.190 8848`
3. 检查认证配置 `NACOS_USERNAME`/`NACOS_PASSWORD`

### 8.3 前端无法访问后端

**问题**：CORS跨域错误或请求超时

**解决**：
1. 检查Vite代理配置是否指向正确的网关地址
2. 确认网关服务已启动
3. 检查防火墙设置

### 8.4 Python服务启动失败（OpenCV）

**问题**：ImportError: libGL.so.1: cannot open shared object file

**解决**：Ubuntu 24.04安装系统依赖
```bash
sudo apt update && sudo apt install -y libgl1 libglib2.0-0 libsm6 libxext6 libxrender-dev
```

Ubuntu 20.04/22.04：
```bash
sudo apt update && sudo apt install -y libgl1-mesa-glx libglib2.0-0 libsm6 libxext6 libxrender-dev
```

### 8.5 模型下载失败

**问题**：模型下载超时或404错误

**解决**：
1. 切换模型源为modelscope：`export PADDLE_PDX_MODEL_SOURCE=modelscope`
2. 手动下载模型到 `/root/.paddlex/official_models/` 目录
3. 设置离线模式：`export PADDLE_PDX_DISABLE_MODEL_SOURCE_CHECK=True`

### 8.6 Python服务启动失败（其他）

**问题**：ModuleNotFoundError

**解决**：
```bash
cd backend/finrag4j-python
uv sync
uv run python main.py
```
