# 47_备份与恢复指南

## 1. 备份策略

### 1.1 数据库备份

| 类型 | 频率 | 保留时间 | 方式 |
| :--- | :--- | :--- | :--- |
| 全量备份 | 每天凌晨2:00 | 7天 | pg_dump |
| 增量备份 | 每小时 | 3天 | WAL日志 |
| 实时备份 | 持续 | - | 主从复制 |

### 1.2 配置备份

- 配置文件：每天备份
- Nacos配置：自动同步
- 密钥文件：离线备份

### 1.3 文件备份

- MinIO存储：跨区域复制
- 日志文件：保留30天
- 代码仓库：Git版本控制

## 2. 备份流程

### 2.1 自动备份脚本

```bash
#!/bin/bash
# 数据库全量备份
BACKUP_DIR="/backup/postgresql"
DATE=$(date +%Y%m%d_%H%M%S)
pg_dump -U postgres finrag4j_auth > "$BACKUP_DIR/auth_$DATE.sql"
pg_dump -U postgres finrag4j_document > "$BACKUP_DIR/document_$DATE.sql"
pg_dump -U postgres finrag4j_search > "$BACKUP_DIR/search_$DATE.sql"
pg_dump -U postgres finrag4j_agent > "$BACKUP_DIR/agent_$DATE.sql"

# 清理7天前的备份
find "$BACKUP_DIR" -name "*.sql" -mtime +7 -delete
```

### 2.2 备份验证

- 备份完成后校验文件完整性
- 定期进行恢复测试
- 验证备份文件可正常恢复

## 3. 恢复流程

### 3.1 数据库恢复

```bash
# 停止相关服务
systemctl stop finrag4j-auth

# 恢复数据库
psql -U postgres finrag4j_auth < /backup/postgresql/auth_20240101_020000.sql

# 启动服务
systemctl start finrag4j-auth
```

### 3.2 恢复步骤

1. 确认需要恢复的数据范围
2. 停止相关服务
3. 从备份恢复数据
4. 验证数据完整性
5. 启动服务
6. 验证业务功能

## 4. 灾难恢复

### 4.1 故障场景

| 场景 | 恢复策略 |
| :--- | :--- |
| 单节点故障 | 切换到备用节点 |
| 数据损坏 | 从备份恢复 |
| 区域故障 | 切换到异地灾备 |

### 4.2 RTO和RPO

- RTO（恢复时间目标）：< 30分钟
- RPO（恢复点目标）：< 1小时
