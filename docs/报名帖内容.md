## 1. 创意名称 + 创意介绍

**创意名称**: FinRag4j - 金融领域RAG智能助手

**想解决什么问题**: 金融从业者在处理海量文档时，难以快速准确地获取所需信息，传统搜索引擎无法理解金融专业术语和上下文语义。

**为什么会想到做这个**: 在金融行业工作中，经常需要查阅大量的研究报告、政策文件、行业分析等资料，信息检索效率低下。大型语言模型虽然强大，但缺乏专业领域的知识，容易产生幻觉。

**大概是什么产品**: 一个面向金融从业者的智能问答系统，基于RAG（检索增强生成）技术，能够精准理解用户问题，从专业文档库中检索相关知识，生成准确可靠的回答。

---

## 2. 目标用户及痛点

**面向哪些用户**:
- 金融分析师、投资经理
- 银行客户经理、风控人员
- 金融行业研究员、实习生

**在什么场景下使用**:
- 研究报告撰写时快速查找资料
- 客户咨询时快速获取产品信息
- 合规审查时检索政策法规
- 日常工作中快速获取专业知识

**当前痛点**:
- 传统关键词搜索无法理解语义，结果不准确
- 需要在多个文档中手动查找信息，效率低下
- 缺乏专业领域知识的AI助手，回答不可靠
- 文档版本管理复杂，难以追踪信息来源

---

## 3. 价值与意义

**效率提升**: 通过智能问答方式，将信息检索时间从小时级缩短到分钟级，大幅提升工作效率。

**商业价值**: 帮助金融机构降低人力成本，提升服务质量，增强竞争力。系统可作为内部知识库，也可对外提供增值服务。

**社会价值**: 促进金融知识的普及和传播，让更多人能够便捷地获取专业金融信息。

## 5. 产品核心功能

| 功能 | 说明 |
|------|------|
| 智能文档解析 | 支持多种格式文档上传，自动解析、分块、向量化存储 |
| 语义检索 | 基于PGVector的向量检索，精准匹配用户意图 |
| RAG问答 | 结合检索结果生成准确回答，提供来源引用 |
| 多知识库管理 | 支持多个知识库独立管理，灵活配置访问权限 |
| 合规检查 | 自动检测回答内容的合规性，降低业务风险 |
| 多轮对话 | 支持上下文记忆，实现连贯的多轮问答体验 |

## 6. 技术架构

| 层级 | 技术 |
|------|------|
| 后端框架 | Spring Boot 3.2 + Spring Cloud 2023 |
| 服务注册/配置 | Nacos 2.2.x |
| RAG框架 | LangChain4j 0.30+ |
| 数据库 | PostgreSQL 16 + PGVector |
| 缓存 | Redis 7.x |
| 消息队列 | RocketMQ 5.0+ |
| 对象存储 | MinIO 2024+ |
| Python服务 | FastAPI + PaddleOCR |
