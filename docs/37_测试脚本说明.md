## 2. 测试脚本说明

### 2.1 01_health_check.py - 健康检查接口测试

**功能说明**：
测试 Python 微服务的 `/health` 健康检查接口，验证服务是否正常运行。

**测试内容**：
- 服务状态（status）
- 服务名称（service）
- 版本号（version）
- Nacos 配置状态

**运行命令**：
```bash
cd tests
python 01_health_check.py
```

**预期结果**：
- HTTP 状态码：200
- 返回 JSON 包含 `status: healthy`

---

**测试内容**：

- 长文本处理

测试 `/api/parse/ocr` OCR 识别接口，验证图片文字识别功能。

### 2.6 06_middleware_check.py - 中间件服务检查

**功能说明**：
检测 Linux 中间件服务器上的所有中间件服务是否正常运行（端口连通性检测）。

**测试内容**：
- Nacos 配置中心 (8848)
- PostgreSQL 数据库 (5432)
- Redis 缓存 (6379)
- MinIO 对象存储 (9000)
- RocketMQ NameServer (9876)
- RocketMQ Broker (10911)
- RocketMQ Dashboard (8088)

**运行命令**：
```bash
cd tests
python 06_middleware_check.py
```

**参数选项**：
```bash
# 使用配置文件 (.env)
python 06_middleware_check.py -c ../deploy/.env

# 快速测试指定IP
python 06_middleware_check.py --host 11.0.1.190
python 06_middleware_check.py --host 192.168.222.186
```

**预期结果**：


运行测试前请确保：
1. Python 微服务已启动：`uv run python main.py`
2. 服务运行在 `http://localhost:8001`

## 6. 故障排查

### 问题：连接失败

**错误信息**：
```
[FAIL] 连接失败：服务可能未启动
```

**解决方案**：
1. 确认 Python 微服务已启动
2. 检查服务端口（默认 8001）
3. 检查防火墙设置

### 问题：OCR 识别失败

**错误信息**：
```
[WARN] OCR识别失败（可能是环境问题）
```

**原因**：
- PaddlePaddle oneDNN 兼容性问题

**解决方案**：
1. 使用支持 AVX2 的 CPU
2. 或安装纯 CPU 版本 PaddlePaddle
3. 或使用 GPU 版本

## 7. 添加新测试

如需添加新的测试脚本，请遵循以下命名规范：

1. 文件名格式：`序号_功能模块_测试名称.py`
2. 序号从 01 开始
3. 脚本应包含主函数 `test_*()`
4. 在 `run_all.py` 中注册新测试

示例：
```python
# 06_new_feature.py
def test_new_feature():
    # 测试代码
    pass

if __name__ == "__main__":
    success = test_new_feature()
    sys.exit(0 if success else 1)
```
