# FinRag4j 配置管理说明

## 文档概述

本文档详细说明 FinRag4j 项目如何管理各个中间件的地址配置，包括配置层级、环境变量管理、Nacos 配置中心使用等。

**文档版本**: v1.0.0  
**编制日期**: 2026-06-22

## 2. 中间件地址配置详解

### 2.1 中间件配置总表

| 中间件 | 默认端口 | 配置项前缀 | 配置来源 | 说明 |
|--------|----------|-----------|----------|------|
| Nacos | 8848/8080 | NACOS_* | .env | 配置中心和服务注册 |
| PostgreSQL | 5432 | POSTGRES_* | .env + application.yml | 关系数据库 |
| Redis | 6379 | REDIS_* | .env + application.yml | 缓存和会话 |
| MinIO | 9000/9001 | MINIO_* | .env + application.yml | 对象存储 |
| RocketMQ | 9876/10911 | ROCKETMQ_* | .env + application.yml | 消息队列 |
| Python服务 | 8001 | PYTHON_* | .env + docker-compose | 文档预处理 |

### 2.2 Nacos 配置

#### 2.2.1 环境变量配置 (.env)

```bash
# Nacos 服务地址
NACOS_HOST=11.0.1.190
NACOS_PORT=8848

# Nacos 3.x 鉴权配置
NACOS_AUTH_ENABLE=true
NACOS_AUTH_IDENTITY_KEY=serverIdentity
NACOS_AUTH_IDENTITY_VALUE=mySecureValue123
NACOS_AUTH_TOKEN=VGhpc0lzTXlTZWNyZXRLZXlXaGljaFNob3VsZEJlMzJDaGFyYWN0ZXJzTGFyZ2U=

# Nacos 命名空间
NACOS_NAMESPACE=public
```

#### 2.2.2 Docker Compose 中的 Nacos 配置

```yaml
nacos:
  image: nacos/nacos-server:v3.2.2
  environment:
    - MODE=standalone
    - NACOS_SERVER_PORT=${NACOS_PORT:-8080}  # 内部端口
    - JVM_XMS=256m
    - JVM_XMX=512m
  ports:
    - "${NACOS_PORT:-8848}:8848"           # 外部端口 8848
    - "${NACOS_PORT_8848:-8848}:8848"       # 外部端口 8848 (重复映射)
    - "${NACOS_PORT_GRPC:-9848}:9848"       # gRPC 端口
    - "${NACOS_PORT_GRPC2:-9849}:9849"      # gRPC 端口2
```

#### 2.2.3 应用中的 Nacos 连接配置

```yaml
# bootstrap.yml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: ${NACOS_HOST:localhost}:${NACOS_PORT:8848}
        namespace: ${NACOS_NAMESPACE:public}
        username: ${NACOS_USERNAME:nacos}
        password: ${NACOS_PASSWORD:nacos}
      config:
        server-addr: ${NACOS_HOST:localhost}:${NACOS_PORT:8848}
        namespace: ${NACOS_NAMESPACE:public}
        file-extension: yml
        shared-configs:
          - data-id: finrag4j-common.yml
            refresh: true
```

### 2.6 RocketMQ 配置

#### 2.6.1 环境变量配置 (.env)

```bash
ROCKETMQ_NAMESRV_CONTAINER_NAME=finrag4j-rocketmq-namesrv
ROCKETMQ_BROKER_CONTAINER_NAME=finrag4j-rocketmq-broker
ROCKETMQ_NAMESRV_HOST=11.0.1.190
ROCKETMQ_NAMESRV_PORT=9876
ROCKETMQ_BROKER_PORT=10911
ROCKETMQ_BROKER_PORT2=10909
```

#### 2.6.2 Docker Compose 中的 RocketMQ 配置

```yaml
rocketmq-namesrv:
  image: apache/rocketmq:5.2.0
  command: sh mqnamesrv
  ports:
    - "${ROCKETMQ_NAMESRV_PORT:-9876}:9876"

rocketmq-broker:
  image: apache/rocketmq:5.2.0
  command: sh mqbroker -n rocketmq-namesrv:9876 --enable-proxy
  ports:
    - "${ROCKETMQ_BROKER_PORT:-10911}:10911"
    - "${ROCKETMQ_BROKER_PORT2:-10909}:10909"
  depends_on:
    rocketmq-namesrv:
      condition: service_healthy
```

#### 2.6.3 应用中的 RocketMQ 配置

```yaml
# application.yml
rocketmq:
  namesrv-addr: ${ROCKETMQ_HOST:localhost}:9876
  producer:
    group: finrag4j-document-producer
```

## 3. 多环境配置管理

### 3.1 环境文件结构

```
deploy/
├── .env                    # 默认配置 (开发环境)
├── .env.devhome           # 家庭开发环境
├── .env.devcompany        # 公司开发环境
├── .env.test              # 测试环境
├── .env.prod              # 生产环境
├── docker-compose.yml
└── docker-compose-base.yml
```

### 3.2 环境配置切换方式

#### 3.2.1 Docker Compose 环境切换

```bash
# 使用默认 .env
docker-compose up -d

# 使用指定环境文件
export COMPOSE_ENV_FILE=.env.test
docker-compose --env-file .env.test up -d

# 或在 .env 文件中指定
echo "COMPOSE_ENV_FILE=.env.test" >> .env
```

#### 3.2.2 各环境配置示例

```bash
# .env.devhome (家庭开发)
NACOS_HOST=localhost

REDIS_HOST=11.0.1.190
```

## 6. 最佳实践

### 6.1 配置管理建议

1. **敏感信息管理**
   - 生产环境密码不要写在 .env 文件中
   - 使用 Kubernetes Secrets 或 Vault 管理
   - Nacos 3.x 启用鉴权

2. **环境隔离**
   - 不同环境使用不同 Nacos 命名空间
   - 使用不同的数据库实例
   - 生产环境禁用 Knife4j

3. **配置版本管理**
   - Nacos 配置发布前先在测试环境验证
   - 保留配置历史版本
   - 重大变更使用灰度发布

### 6.2 常见问题排查

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 服务无法注册到 Nacos | Nacos 未启动或网络不通 | 检查 Nacos 健康状态和网络连通性 |
| 配置不生效 | 配置未发布或格式错误 | 检查 Nacos 配置内容和格式 |
| 服务间调用失败 | 服务未注册或地址错误 | 检查服务注册列表和地址配置 |
| 热更新不生效 | 未启用 refresh 或未添加注解 | 启用 refresh: true，使用 @RefreshScope |
